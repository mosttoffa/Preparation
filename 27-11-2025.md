## 
<b>Async/Await (অ্যাসিঙ্ক/অ্যাওয়েট) : </b><br> 
Async/Await (অ্যাসিঙ্ক/অ্যাওয়েট) কী? <br> 
C# এ async/await হলো এমন feature যা দিয়ে কোড background এ non-blocking ভাবে কাজ করতে পারে, আর main thread বা UI/Request flow আটকে রাখে না। সহজভাবে, এটা দিয়ে আপনি slow কাজগুলোকে অপেক্ষা করাতে পারেন কিন্তু পুরো অ্যাপ বন্ধ না করে। <br> 
কেন use হয়? <br> 
Fast response: Server request আটকে থাকে না.  UI freeze প্রতিরোধ: Web/UI thread stuck হয় না.  scalability: Enterprise app একসাথে বেশি request handle করতে পারে.  efficient resource use: Thread idle বসে wait করে না. <br> 
কখন use হয়? <br> 
 Database query / complex stored procedure call.  File read/write.  API/HTTP call.  Image/Report/PDF generation.  Email/SMS send.  Realtime signal/event-based response processing.  <br> 
কখন use করবেন? <br> 
যখন কাজটি সময় নেয়.  যখন আপনি অপেক্ষা করছেন (network, IO, DB, external service).  যখন controller থেকে response দ্রুত ফিরে দিতে চান, কিন্তু result পরে আসবে. <br> 
Syntax / Structure : 
```
public async Task<ReturnType> MethodName(Parameter)
{
    // await দিয়ে slow task call করবেন
    ReturnType result = await SomeSlowTask();
    return result;
}
Rules মনে রাখুন: 
Method এর আগে async keyword লাগে.  Slow task call এর আগে await লাগে.  Return type সাধারণত Task বা Task<T> হয়.  Calling side থেকেও await দিয়ে call করতে হয়. 
```
LINQ তে async এর usecase : <br>
DB বা service থেকে list fetch করে LINQ run করা: <br> 
```
var list = await service.GetProductsAsync();
var result = list.Where(x => x.UnitPrice > 100).ToList();
```





2. Delegates (ডেলিগেট) : <br>
ধারণা: ডেলিগেট হলো একটি টাইপ-সেফ ফাংশন পয়েন্টার (Type-Safe Function Pointer)। এটি একটি মেথডকে রেফার করে। আপনি ডেলিগেট ব্যবহার করে একটি মেথডকে ভেরিয়েবলের মতো পাস করতে পারেন অথবা অন্য মেথডের আর্গুমেন্ট হিসেবে পাঠাতে পারেন। ইভেন্ট হ্যান্ডলিং এবং ল্যাম্বডা এক্সপ্রেশন (Lambda Expressions) এর মূল ভিত্তি হলো ডেলিগেট। <br>

3. Events (ইভেন্ট) :
ধারণা: ইভেন্ট হলো এমন একটি প্রক্রিয়া যা একটি ক্লাসকে (Publisher) অন্য ক্লাসগুলিকে (Subscriber) কোনো কিছু ঘটার বিষয়ে জানাতে সাহায্য করে। এটি মূলত ডেলিগেটের উপর ভিত্তি করে তৈরি। এটি ডিকাপলিং (Decoupling) নিশ্চিত করে, অর্থাৎ Publisher কে জানতে হয় না কে বা কারা তার ইভেন্টটি শুনছে। <br>
